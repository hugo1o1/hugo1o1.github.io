clean code #

据统计，阅读代码的时间 和 写代码的时间比例大致是10:1

平时拿到一个新的需求，看完需求文档之后，接着要做的事情就是读上个迭代的代码，有时候这是非常痛苦的，比如命名是中文的，命名的单词是错误的，看到一个啥都干的函数，这个函数在各个地方调用...

如何去评价代码质量？就是你说“我*”（wtfs） 每分钟的次数。
